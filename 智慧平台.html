01 計畫目標 - 提升服務效能

核心目標

透過 AI 應用服務，更快速、準確地回應和處理民眾需求，提供 7x24小時 持續諮詢及分流服務，提升為民服務品質。

主要任務

  • 導入網頁式文字生成智慧客服。
  • 建立統一管理平台系統。
  • 客製化功能開發與維護增修。
  • 提供明確途徑轉接至真人客服。

知識庫建置

本案將進行各單位知識庫盤點、建置與訓練，並將QA問答進行標準化與正規化，作為智慧客服回應的基礎。

工作內容 - 知識庫建立與優化

知識庫建立

  • 建置方法： 匯入各單位FAQ、爬取網頁資料、協助新增資料。
  • 內容設計： 針對專有名詞、用戶用詞進行語意分析，設計情境式、多輪式對話。
  • 回應風格： 透過生成式AI設定親切、務實等特定語氣。

數量與正確率目標

  • 知識點數量： 上線時至少 1,000 點，全期至少 2,500 點。
  • 多輪式對話： 上線時至少 10 組，全期至少 30 組。
  • 正確率要求：
    • 上線時：> 80%
    • 上線後三個月：> 85%
    • 後續維護：> 95%

工作內容 - 知識庫維護與測試

維護與管理

  • 權限劃分： 單位承辦人具編輯權限，單位主管具審核權限。
  • 更新機制： 規劃知識庫更新SOP，定期審核、更新及擴充。
  • 定期檢討： 定期召開會議，檢討FAQ正確率、覆蓋率等指標。
  • 內網知識庫： 區分內外部使用者資料，確保資訊安全。

知識庫測試方式

測試需各單位協助判斷AI回答狀況，並將其結果回饋於測試表單中，流程如下：

  1. 建立初步FAQ
  2. 設計測試題 (Prompt)
  3. 各單位進行提問測試
  4. 收集測試回饋
  5. 錯誤分析與模型微調
  6. 完成調校後上線

工作內容 - 服務水準協議 (SLA)

服務水準指標 (SLI)

系統可用性 反應時間 故障次數 負載人數 正確率 覆蓋率 問題解決率 真人客服轉接率

服務水準目標 (SLO)

  • 系統可用性： 故障發生後 8 小時內 提出解決方案或替代方案。
  • 反應時間： 平均回應時間在 3 秒內 (比率達 95%以上)。
  • 故障次數： 每月系統無法作業時間累計不得超過 24 小時，且每月不得超過 1 次。
  • 負載能力： 系統須同時容許至少 200 人上線。

方案說明 - 方案差異比較

方案一：混合雲架構 (建議方案)

兼顧資安、效能與成本效益，核心系統與訓練機落地，彈性運用雲端LLM。

  • 雲端LLM服務: AOAI / AWS / Google Gemini
  • 主系統: 本部VM (Web/AP/DB 各*2)
  • NLP 模型推論機: 本部VM
  • NLP 模型訓練機: 本部實體機 (GPU: L40S)

方案二：混合雲 - 高效能推論

將推論機改為實體GPU主機，提升即時回應效能，成本較高。

  • 雲端LLM服務: AOAI / AWS / Google Gemini
  • 主系統: 本部VM (Web/AP/DB 各*2)
  • NLP 模型推論機: 本部實體機 (GPU: RTX6000/L40S)
  • NLP 模型訓練機: 本部實體機 (GPU: L40S)

方案三：純雲端架構

所有系統皆部署於微軟雲端，建置快速但長期營運成本最高，且有機敏資料上雲考量。

  • 雲端LLM服務: AOAI
  • 主系統: 微軟雲端方案
  • NLP 模型推論機: 微軟雲端方案
  • NLP 模型訓練機: 微軟雲端方案

方案說明 - 大型語言模型(LLM)選擇

OpenAI GPT-4

透過 Azure OpenAI (AOAI) 服務

  • 優勢： 強大的自然語言生成能力、深度微調能力，可提供更自然的交互體驗。
  • 資安： AOAI提供企業級防護機制，包含資料加密、身份驗證與隱私保護，使用者數據不會被用於模型訓練。

Google Gemini 2.5 Pro

透過 Google Cloud 服務

  • 優勢： 整合多模態資料能力強，與Google生態系統整合性佳，利於進行深入資料分析。
  • 資安： 利用Google Cloud IAM提供精細權限控制，所有傳輸與儲存的數據均為加密狀態。

附錄 - 資通安全要求

法規遵循與檢測

  • 遵循「資通安全管理法」及相關規範。
  • 每半年執行 1 次弱點掃描。
  • 每年執行 1 次滲透測試。
  • 提供「系統安全檢測及修補報告」。

個資與AI安全

  • PII 保護： 導入個人可識別資訊(PII)自動偵測與遮罩技術。
  • AI安全： 需考量OWASP Top 10 for LLM，防止資料洩漏及輸出失真/誤導。

附錄 - 雲端LLM防護機制

各大雲端平台皆提供完善的企業級安全防護機制，以 Azure OpenAI (AOAI) 為例：

資料加密機制

  • 靜態數據加密： 儲存於 Azure 的所有數據均使用 AES-256 加密。
  • 傳輸數據加密： 使用 TLS 1.2/1.3 加密協議保護傳輸中的數據。

身份驗證與存取控制

  • Azure AD 整合： 支援MFA多因素驗證與RBAC角色型存取控制。
  • 條件性存取： 可依據IP、設備等條件限制存取。

使用者數據隱私與保護

Microsoft 確保 AOAI 所處理的應用數據和查詢 不會被用於後續的模型訓練，確保業務數據的隱私性。
